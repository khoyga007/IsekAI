IsekAI — progress notes (last updated 2026-06-12)
====================================================

For the full picture, read HANDOFF.md. This is just a quick scratchpad for
"where we are right now" and "what to pick up next".

DONE (2026-06-13)
-----------------
[x] AniList avatar matching fixed — was effectively dead. AniList returns
    full names in given-name-first order ("Luffy Monkey", "Zoro Roronoa")
    while LLM bibles write conventional order ("Monkey D. Luffy"), so the
    exact/substring/last-name matcher never hit. matchCanonAvatar now
    tokenizes (lowercase, punctuation stripped) and matches order-
    insensitively: token-set equality → subset either way → shared
    distinctive token (≥3 chars). Verified live: One Piece + AoT casts
    all match, OC names return null (sigil fallback).

DONE (2026-06-12)
-----------------
[x] Continue to Part 2 / sequel campaign flow. Adds topbar fast-forward
    drawer (`SequelView`) that generates editable recap, stores it on a fresh
    Part N+1 campaign, keeps bible/protagonist/HUD state, clears scenes/
    crystals/bookmarks, links `prevId`, and injects recap into stable prompt.
    Follow-up fix: first turn of a sequel now uses a continuation-opening
    prompt instead of generic new-campaign prologue. i18n EN/VI added.
    Also kept from same pass: <bible-add> tag (AI can add recurring
    characters/factions/rules to the bible mid-game) and healCompactFormat
    (heals compact-format output back to XML for the parser).
[x] Story Mode selector (Adventure/Romcom/Romance/Harem) REMOVED — tried it,
    this UI doesn't fit romance pacing. Romance will be a separate app later.
[x] Untagged-prose fallback now splits into manga-sized panels. When the
    model drifts off XML (mimics compact history format), parseStory used to
    dump everything into ONE giant narration panel ("wall of text" blocks).
    pushNarrationChunks splits on blank lines, then regroups long paragraphs
    into ~3-sentence panels (max ~320 chars). Also applies to over-long
    single tags: <narrate> >320 chars, <say>/<think>/<system> >400 chars
    (split chunks keep the same speaker → consecutive same-speaker bubbles,
    manga style). <act> never split — stage directions read as one unit.
[x] DeepSeek V4 Flash "no text" fix (4 parts). Root cause: v4-flash defaults
    to THINKING mode; CoT streams via delta.reasoning_content (was dropped)
    and counts against max_tokens (1200 got eaten whole on long contexts →
    empty content). Fixes: (1) openai-compat yields reasoning chunks +
    90s stall watchdog on data events; (2) playTurn maxTokens 1200→4096;
    (3) parseStory routes untagged text into narration panels instead of
    discarding (also strips partial trailing tag); (4) playTurn throws on
    empty output instead of committing empty scene. DeepSeek defaultModels
    updated to V4 chips (deepseek-chat alias deprecated 2026-07-24).
[x] Removed AI image generation (Nano Banana 2 + Pollinations.ai) to keep
    the app strictly free to run. Avatars are now AniList canon match +
    procedural sigil only. engine/avatars.ts slimmed to AniList fetch/match,
    AvatarPicker is AniList-only (canon match pre-selected), Onboarding
    skips the picker entirely for non-canon sources (world/rng) since
    there is nothing to pick. Old gen code lives in git history.

KNOWN BUGS / WATCHLIST
----------------------
- AvatarPicker only shows protagonist when bible.keyCharacters is empty.
  AI sometimes skips key characters (saw on "Owari no Seraph" with OC
  protagonist). Could be fixed by tightening BIBLE_SYS prompt to
  mandate at least 3 key chars even when protagonist is OC.
- Retry button does NOT roll back HUD ops (HP delta would apply twice).
  Pre-existing limitation; documented but not fixed.

DESIGN NOTES (carried from session discussion)
----------------------------------------------
- App identity: "manga writing studio" — hybrid between game (AI Dungeon)
  and writing tool (Sudowrite). User confirmed hybrid is the sweet spot.
  Rule of thumb when adding features: do they serve play OR craft? If
  both → high priority. Edit/retry affordances are gated behind hover/
  confirm so game-feel isn't disrupted.
