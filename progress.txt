DONE (2026-06-13)

[x] Streaming perf pass (Yang: "tối ưu cực hạn") — kills long-turn jank:
    1. rAF coalesce (InputBar.runTurn): onDelta was parseStory(acc) + new
       draft-panel array on EVERY SSE chunk (hundreds/turn) = O(n²)
       re-parse + a full render per chunk. Now the newest acc is held and
       a single requestAnimationFrame flush parses+renders once per frame
       (~60fps). Cancelled on stream end + on error/abort; final parse
       after await is unchanged so the committed result stays exact.
    2. memo(ScenePage) (StoryView): the view re-renders each frame while
       the draft grows; committed scenes pass only stable-ref props, so
       React.memo lets the whole committed-scene subtree skip re-render
       (the bulk of the DOM on long campaigns). Draft/reveal scene passes
       fresh callbacks → falls through and updates as needed.
    3. retrieval.ts archiveIndex cache: tokensPerScene + document-freq map
       were rebuilt every turn (O(archived × tokens)). Now cached, keyed
       by the archived id set — recomputes only when compression grows the
       archive (~every 8 turns). Archived scenes are read-only so tokens
       can't go stale.
[x] Toy batch (Yang: "múc hết cho anh") — 5 items shipped:

       (Bounty / Hero Rank / Cursed Energy Grade / Power Level...), seeded
       at the protagonist's standing. (closes the OPEN canon-rank item.)
    2. Keyboard shortcuts (InputBar global keydown, window CustomEvents):
       Esc = skip typewriter ("isekai:skip"), R = retry, Ctrl/Cmd+Z =
       undo, Ctrl/Cmd+B = bookmark ("isekai:bookmark"). Listeners in
       StoryView (skip) + App (bookmark). Guards typing fields, requires

    3. Manga combat SFX: action panels in combat-mood scenes stamp a
       rotated hand-drawn onomatopoeia (ドンッ / RẦM! / KRẮC!...), hash-
       seeded per panel so it's stable. mood threaded ScenePage →

    4. RecapModal "Previously on...": reopening a campaign after >24h

    5. StatsView studio drawer (new rail item, BarChart3): turns, story/
       player word counts, crystals, cast, archived count + beat
       distribution bars. All computed from the campaign on open, no
       tracking/persistence. i18n EN+VI added (rail.stats, stats.*).

[x] Memory upgrade PHASE 2 SHIPPED: lexical RAG over archived scenes
    (engine/core/retrieval.ts — do NOT re-implement). Per turn, query =
    player input + latest live scene; archived scenes scored by token
    overlap. Signal filtering: bible-known name tokens weigh 3x (but a
    name present in EVERY archived scene — the protagonist — counts 0);
    common tokens count only if rare across the archive (document
    frequency <= 25% of scenes), because the query is hundreds of tokens
    and raw overlap would match everything. Threshold keeps quiet turns
    at zero injection. Top scenes (max 5, ~1000 est. tokens) injected
    verbatim into the dynamic block as "RECALLED SCENES", chronological.
    No embeddings, no API, fully local. VERIFY IN PLAY: NPC absent 30+
    turns, mention name → their archived scenes return; generic input →
    no injection.

[x] Anthropic 2nd cache breakpoint fix (providers/anthropic.ts). Was in
    KNOWN BUGS: hoisting ALL system messages into the `system` param put
    the volatile dynamic block (HUD/crystals) between the stable block
    and history in the real prefix → history breakpoint missed whenever
    HUD changed (1.25x write instead of 0.1x read). Now only LEADING
    system messages are hoisted; mid-conversation system messages fold
    into the next user message, preserving true prefix order. Matters
    now that TOKEN_BUDGET is 24k — long Claude campaigns would have paid
    full input price almost every turn.
[x] Long-term memory upgrade, phase 1 of 2 (Yang approved roadmap):
    (1) TOKEN_BUDGET 5500 → 24000 (contextManager.ts). Forgetting started
    around turn ~15-20 — source of most consistency complaints. Prefix
    caching keeps the cost increase small.
    (2) Scene archiving. compressIfNeeded now MARKS compressed scenes
    archived:true instead of deleting them. Model context excludes them
    (playTurn history filter; crystal still represents them) — zero token
    change. StoryView collapses the archived prefix behind one divider
    ("N archived turns — GM remembers these only as a summary", i18n
    EN/VI), expandable read-only. Doubles as cheap virtualization (Ariel
    handoff #4 — largely covered now). Scenes already deleted by old
    compression are unrecoverable. types.ts, contextManager.ts,
    storyEngine.ts, StoryView.tsx, i18n.ts.
[x] Crystal/auto-evo audit fixes (post-Ariel-540bea3 cleanup):
    (1) <crystal> removal finished — rule 7 in STORY_SYS still taught the
    tag after its syntax line was removed (model saw the name, not the
    syntax → malformed output risk). Fully removed; parser keeps the tag
    for back-compat, compression is now the only crystal source.
    (2) ScenePage "TURN NN" label used array index, wrong after context
    compression drops old scenes — now uses absolute s.turn (sceneIdx
    stays array-based for edits). Same class as Ariel's TopBar fix.
    (3) applyBibleAdds dedupe: identical (Update: ...) no longer appended
    repeatedly (models re-emit the same bible-add for turns; every desc
    change invalidates the stable-prompt cache), duplicate rules skipped.
    (4) bible name matching now tokenized order-insensitive ("Rinka" ==
    "Rinka Akatsuki") instead of exact equality — was creating duplicate
    entries. promptBuilder.ts, StoryView.tsx, streamParser.ts.
[x] Scene.turn backfill for old saves (campaign.ts healCampaign). Absolute
    turn numbering (540bea3) reads `scenes[last].turn + 1`, but campaigns
    saved before the field existed have no `turn` → NaN in TopBar chapter,
    bookmark labels, crystal pins, commitTurn. healCampaign now backfills
    turn=index when missing.
[x] Streaming-read UX overhaul (StoryView.tsx). Three interacting bugs:
    (1) auto-scroll yanked view to bottom on every chunk even while user
    was reading above; (2) ALL draft panels typewrote in parallel so later
    panels spoiled the scene; (3) commitTurn at stream end swapped draft
    for committed scene with typewriter off → remaining text popped in
    full. Fixes: near-bottom-gated follow scroll (ResizeObserver on
    content + scroll listener; detaches when user scrolls up >160px from
    bottom, re-attaches at bottom); sequential panel reveal (ScenePage
    typedCount gates visiblePanels — panel N+1 mounts only after N types
    out, PanelContent fires onTyped); reveal continues across commit
    (revealId state + shared key="reveal" between draft and committed-
    reveal ScenePage preserves useTypewriter idxRef so no restart/pop;
    chips appear only after reveal completes). Click-to-skip still
    reveals everything instantly.
[x] Suggestion chips disappearing on long campaigns fixed. Compact history
    has no representation for <scene/>/<suggest>, so after many turns the
    model imitates history and drops the closers (also lost entirely on
    untagged drift / output truncation). Added an always-on FORMAT REMINDER
    section to buildSystemPromptDynamic — sits after history, right before
    user input, where recency is strongest. Side effect: dynamic block is
    never empty anymore (~40 tokens/turn, fine on DeepSeek message-order
    caching). storyEngine.ts.
[x] Narration misrendered as anonymous "thought" bubble fixed. Two paths
    produced a speakerless <think> panel: (1) model emits <think> without
    speaker attr around third-person prose; (2) healCompactFormat turns
    *asterisk emphasis* in untagged drift output into <think> with no
    speaker. Format mandates speaker="Name" on every thought, so toPanel
    now reroutes speakerless <think> to a narration panel. Also tightened
    STORY_SYS: <think> = first-person only, speaker REQUIRED. storyEngine.ts.
[x] "Rendered more hooks than during the previous render" crash fixed.
    REAL root cause: PanelContent called useSpeakerAvatar conditionally —
    only inside the `dialogue` branch, after early returns. Panels are
    keyed by index and a draft panel can change KIND between stream
    chunks (re-parse + splitting), so the same instance rendered 1 hook
    (narration) then 2 hooks (dialogue) → crash. Hoisted the hook above
    the early returns. Also removed AnimatePresence around panels.map
    (no exit animations; index keys + reshuffling risked double-mounts).
    StoryView.tsx.
[x] Anthropic 401 on browser/webview calls fixed — added required
    "anthropic-dangerous-direct-browser-access: true" header to
    providers/anthropic.ts. Without it Anthropic rejects all CORS
    requests with 401 authentication_error.
[x] AniList avatar matching fixed — was effectively dead. AniList returns
    full names in given-name-first order ("Luffy Monkey", "Zoro Roronoa")
    while LLM bibles write conventional order ("Monkey D. Luffy"), so the
    exact/substring/last-name matcher never hit. matchCanonAvatar now
    tokenizes (lowercase, punctuation stripped) and matches order-
    insensitively: token-set equality → subset either way → shared
    distinctive token (≥3 chars). Verified live: One Piece + AoT casts
    all match, OC names return null (sigil fallback).

    (split chunks keep the same speaker → consecutive same-speaker bubbles,
    manga style). <act> never split — stage directions read as one unit.

    to THINKING mode; CoT streams via delta.reasoning_content (was dropped)

    empty content). Fixes: (1) openai-compat yields reasoning chunks +
    90s stall watchdog on data events; (2) playTurn maxTokens 1200→4096;
    (3) parseStory routes untagged text into narration panels instead of
    discarding (also strips partial trailing tag); (4) playTurn throws on
    empty output instead of committing empty scene. DeepSeek defaultModels
    updated to V4 chips (deepseek-chat alias deprecated 2026-07-24).

DESIGN NOTES (carried from session discussion)
----------------------------------------------
- App identity: "manga writing studio" — hybrid between game (AI Dungeon)
  and writing tool (Sudowrite). User confirmed hybrid is the sweet spot.
  Rule of thumb when adding features: do they serve play OR craft? If
  both → high priority. Edit/retry affordances are gated behind hover/
  confirm so game-feel isn't disrupted.
